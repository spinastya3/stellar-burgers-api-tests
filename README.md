# Автотесты для API сервиса Stellar Burgers (Дипломный проект)
Данный проект содержит автоматизированные тесты для REST API сервиса Stellar Burgers. Тесты охватывают основные бизнес-процессы: управление пользователями, авторизацию и работу с заказами.

| Технология | Описание |
| :--- | :--- |
| **Java 11** | Основной язык разработки |
| **REST Assured** | Библиотека для автоматизации тестирования REST API |
| **JUnit 4** | Тестовый фреймворк для организации и запуска тестов |
| **Allure** | Инструмент для построения детальных и понятных отчетов |
| **Maven** | Система сборки и управления зависимостями |
| **Lombok** | Библиотека для сокращения бойлерплейт-кода (POJO модели) |
| **JavaFaker** | Генерация случайных валидных данных для тестов |

##  Что покрыто тестами

### 1. Создание пользователя (`POST /api/auth/register`)
*   Успешное создание уникального пользователя.
*   Ошибка при создании пользователя, который уже зарегистрирован.
*   Ошибка при регистрации с незаполненным обязательным полем (использована параметризация).

### 2. Авторизация (`POST /api/auth/login`)
*   Вход под существующим (валидным) пользователем.
*   Ошибка авторизации при вводе неверного логина или пароля.
*   Обработка кейсов с пустыми значениями и отсутствующими полями в запросе.

### 3. Создание заказа (`POST /api/orders`)
*   Успешное создание заказа авторизованным пользователем.
*   Создание заказа неавторизованным пользователем (проверка прав доступа).
*   Валидация списка ингредиентов: создание с ингредиентами, без них и с неверным хешем.

## Архитектурные особенности
*   **BaseClient & RequestSpecification:** Реализована централизованная конфигурация RestAssured (Base URL, Headers, Logging) для исключения дублирования кода.
*   **Clean Data Policy:** В проекте реализован механизм автоматической очистки данных. В `BaseTest` настроен цикл `tearDown`, который после каждого теста удаляет созданных пользователей по их `accessToken`.
*   **Service Object Pattern:** Логика запросов вынесена в Client-классы (`UserClient`, `OrderClient`), что делает тесты читаемыми и легкими в поддержке.
*   **Allure Integration:** Все действия в клиентах обернуты в аннотации `@Step`. В отчеты автоматически добавляются логи запросов и ответов (фильтры настроены в `BaseClient`).

##  Запуск проекта

### Прогон всех тестов
```bash
mvn clean test
```

### Генерация Allure-отчета
Для локального просмотра интерактивного отчета в браузере:
```bash
mvn allure:serve
```
